Type Definitions

typedef

ColumnReference

A column in Grid can be referred by the following values:
number : Column index including all invisible (hidden) columns
string : Field name of the column
ColumnDefinition : Column definition object
Type:
number | string | ColumnDefinition
typedef

GridOptions

Configuration object that can be provided directly at the initialization phase
Type:
Object
Properties:
Name Type Attributes Default Description
columns Array.<(ColumnDefinition~Options|string)> <optional>
 Collection of the column definitions
defaultColumnOptions ColumnDefinition~Options <optional>
 All Columns will inherit from this default object
rows Array.<RowDefinition~Options> <optional>
 Collection of the row definitions
fields Array.<string> <optional>
 Shorthand for setting fields
rics Array.<string> <optional>
 Shorthand for creating row definition
plugins Array <optional>
 Additional plugins for core grid (from getBlotter()/getCoreGrid())
extensions Array <optional>
 Alias to `plugins`
scrollbar boolean <optional>
 true Enable both vertical and horizontal scrollbar in grid
linearWheelScrolling boolean <optional>
 false If enabled, wheel scrolling will move the content linearly regardless of how long the content is (i.e. no skipping).
stepScroll boolean <optional>
 false If enabled, a full row will be scrolled at a time instead of actual scroll value.
pageSize string <optional>
 0 Make grid show specified number of row
rowHeight number <optional>
 32 Set default row height for body (content) section
headerRowHeight number <optional>
 32 Set default row height for header section
noColumnDragging boolean <optional>
 false All columns can be moved by dragging. If this option is true, the feature is disabled
columnReorder boolean <optional>
 true Alias with noColumnDragging but in opposite value. This property deprecated in favor of noColumnDragging.
rowHighlighting boolean <optional>
 true Highlight a row when hovering mouse over grid
autoLayoutUpdate boolean | number <optional>
 false Grid will check for layout change periodically.
columnFreezing number <optional>
 Enable horizontal scrollbar after `columnFreezing` index
freezeColumn number <optional>
 An alias to columnFreezing
pinnedRightColumns number <optional>
 0 Number of columns to be frozen/pinned on the right side.
staticDataRows Array <optional>
 Shorthand for calling Grid#addStaticDataRows
dataFields Array.<string> <optional>
 Shorthand for calling Grid#addDataFields
rowVirtualization boolean <optional>
 true If disabled, all data rows will be rendered. This will greatly impact grid's performance if the data set is huge.
columnVirtualization boolean <optional>
 false If enabled, all columns will be rendered. This will greatly impact grid's performance if the column set is huge.
topFreezingCount number | null | boolean <optional>
 null If number >= 0 will fix number of frozen title section, If false = disabled scrollbar, if null then title section will freeze auto when new section added, this option will not work with scrollbar option.
bottomFreezingCount number | null <optional>
 null If number >= 0 will fix nuber of frozen footer section, if null then footer section will freeze auto when new section added, this option will not work with scrollbar option.
borders boolean <optional>
 true Lines around grid element
gridlines boolean <optional>
 true Horizontal and Vertical lines for ONLY content sections
verticalLines boolean <optional>
 true Vertical lines for all sections
horizontalLines boolean <optional>
 true Horizontal lines for all sections
RTK * <optional>
 null rtk toolkit instance
synapse Grid~SynapseConfig <optional>
 null synapse config object
contentBottomPadding number <optional>
 0 Padding that is added below the last section. The padding is still a part of scrollable content.
dataConflationRate number <optional>
 0 set delay for data to avoid to much sorting operation when data changed
dataComposed function <optional>
 Handler for dataComposed event
typedef

RowReference

A row in Grid can be referred by the following values:
number : Row index of a visible rows, excluding invisible (hidden) rows
RowDefinition : Row definition object
string : Unique row Id existing in current data table, including invisible rows
Type:
number | string | RowDefinition

addStaticDataRows(dataRows, fieldsopt)

Add multiple rows without `ric` property in RowDefinition
Parameters:
dataRows
Array
2D Array of data or Array of Field/Value pair object
fields
Array.<string>
<optional>
If no field is given, field defined in column definition will be used instead
See:
  • Grid#logDV
  • Grid#logDT
Example:
var grid = new tr.rt.Grid(grid_div, options);
grid.addStaticDataRows([
 [10, "a", true],
 [20, "b", false],
 [30, "c", true]
], ["field1", "field2", "field3"]); // 3 rows are appended


grid.addStaticDataRows([ // Object format can also be used
 {"field1": 40, "field2": "a",  "field3": true},
 {"field1": 50, "field2": "b"}, // each row doesn't have to be the same
 {"field1": 60, "field4": "some thing"}
]); // 3 more rows are appended
grid.logDV({"colNames": ["field1", "field2", "field3", "field4"]}); // See results in console
